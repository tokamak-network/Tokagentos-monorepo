## Why Tokagent Cloud?

You built an agent. Now you need it running 24/7, accessible from anywhere, with proper monitoring.

The traditional path: write a Dockerfile, configure cloud infrastructure, set up load balancing, create CI/CD pipelines, manage SSL certificates, configure health checks...

**With Tokagent Cloud: two commands.**

```bash
tokagentos login
tokagentos deploy --project-name my-agent
```

That's not a simplified example. That's the entire deployment process.

## Customize Your Deployment

The defaults work for most agents. When you need more control:

### Add Environment Variables

```bash
tokagentos deploy \
  --project-name my-agent \
  --env "OPENAI_API_KEY=sk-xxx" \
  --env "DATABASE_URL=postgresql://..."
```

### Scale Resources

```bash
tokagentos deploy \
  --project-name my-agent \
  --cpu 512 \
  --memory 1024 \
  --desired-count 2  # Multiple instances
```

| Option            | Default | Description                    |
| ----------------- | ------- | ------------------------------ |
| `--cpu`           | 1792    | CPU units (1792 = 1.75 vCPU)   |
| `--memory`        | 1792    | Memory in MB (1792 = 1.75 GiB) |
| `--desired-count` | 1       | Number of container instances  |
| `--port`          | 3000    | Container port                 |

### Update Your Agent

Same command, zero downtime:

```bash
tokagentos deploy --project-name my-agent
```

The CLI detects it's an update and rolls out changes without interruption.

## Troubleshooting

### Login Issues

**"Browser didn't open"**

```bash
tokagentos login --no-browser
```

**"Authentication timed out"**

```bash
tokagentos login --timeout 600
```

**"API key not found after login"**

- Ensure you ran `tokagentos login` from your project directory
- Check `.env` for `TOKAGENTOS_CLOUD_API_KEY`
- Verify the key starts with `tokagent_`

### Deployment Issues

**"Docker not running"**

Start Docker Desktop, then verify with `docker info`.

**"API key invalid"**

- Check you copied the full key (starts with `tokagent_`)
- Verify: `echo $TOKAGENTOS_CLOUD_API_KEY`

**"Build failed"**

- Ensure your project has a `Dockerfile`
- Check Docker has enough resources (Settings → Resources)
- Try: `docker build .`

**"Deployment stuck"**

- Check logs: `tokagentos containers logs --project-name my-agent`
- Verify you have credits in your account

## CLI Reference

### Authentication

| Command                       | Description                  |
| ----------------------------- | ---------------------------- |
| `tokagentos login`               | Authenticate (opens browser) |
| `tokagentos login --no-browser`  | Authenticate without browser |
| `tokagentos login --timeout 600` | Custom timeout (seconds)     |

### Deployment

| Command                                | Description                        |
| -------------------------------------- | ---------------------------------- |
| `tokagentos deploy --project-name <name>` | Deploy with project name           |
| `tokagentos deploy --api-key <key>`       | Deploy with explicit API key       |
| `tokagentos deploy --port <port>`         | Set container port (default: 3000) |
| `tokagentos deploy --cpu <units>`         | Set CPU units (default: 1792)      |
| `tokagentos deploy --memory <mb>`         | Set memory in MB (default: 1792)   |
| `tokagentos deploy --desired-count <n>`   | Set instance count (default: 1)    |
| `tokagentos deploy --env "KEY=VALUE"`     | Add environment variable           |
| `tokagentos deploy --skip-build`          | Skip Docker build                  |
| `tokagentos deploy --platform <arch>`     | Set Docker platform                |

### Container Management

| Command                              | Description                 |
| ------------------------------------ | --------------------------- |
| `tokagentos containers list`            | List all deployments        |
| `tokagentos containers list --json`     | List as JSON                |
| `tokagentos containers logs`            | View logs                   |
| `tokagentos containers logs --follow`   | Follow log output           |
| `tokagentos containers logs --tail <n>` | Show last N lines           |
| `tokagentos containers delete`          | Delete deployment           |
| `tokagentos containers delete --force`  | Delete without confirmation |

## Environment Variables

The CLI looks for API keys in this order:

1. `--api-key` flag (if provided)
2. `TOKAGENTOS_API_KEY` environment variable
3. `TOKAGENTOS_CLOUD_API_KEY` environment variable
4. `TOKAGENTOS_CLOUD_API_KEY` in project `.env` file

The `tokagentos login` command writes to `TOKAGENTOS_CLOUD_API_KEY` in your project's `.env` file.
