No browser wallet?

The dashboard requires an EIP-1193 provider (most browser wallets inject one as window.ethereum). Without one, you can still call the proxy directly via curl following the steps in the project's landing page Coding agents section.

How x402 works (open to read)

x402 is the open HTTP-native payment protocol — pay per call, no API keys to manage. When this agent calls another agent or paid endpoint, the server can demand payment by returning 402 Payment Required. The client signs a one-shot EIP-3009 voucher and retries with X-Payment.

  1. Request — agent sends a normal HTTP call.
  2. 402 challenge — server replies with payment requirements: asset, amount, recipient, validity window, nonce.
  3. Cap check — local middleware checks the request against X402_MAX_PAYMENT_PER_CALL_PTON and X402_MAX_TOTAL_SPEND_PTON. Over cap → refuse with a clear error, never sign.
  4. Sign + retry — operator wallet signs an EIP-3009 TransferWithAuthorization; agent retries with the base64 voucher in X-Payment.
  5. 200 receipt — server accepts, response unblocks. The receipt is emitted as X402_PAYMENT for observability.

The same operator wallet that holds your PTON top-up signs outbound vouchers. You don't need a separate identity — your top-up balance funds both inbound gateway calls (your bot's chat) and outbound calls to other paid agents.

Install this key into your local .env?

This will write BILLING_CHAT_KEY=<key> into the .env in your project root and restart the agent so the new key takes effect. The chat will be unavailable for a few seconds while it relaunches.

If your agent isn't running under a supervisor (the standard bun dev setup), you may need to relaunch it manually.
