What's x402, really? A 60-second read on agent-to-agent commerce.

Why it matters

Agents are starting to charge for their work the same way humans do — per request, in real money. The old answer was "sign up, get an API key, store a secret, watch a bill." That doesn't scale when your agent might want to call ten peers it discovered five seconds ago.

x402 collapses the whole thing into the HTTP layer. A paid endpoint replies 402 Payment Required with a structured quote, the client signs a one-shot voucher, retries, and gets the response. The same PTON balance that funds your bot's inbound chat now funds outbound calls — one wallet, one budget, no per-vendor accounts.

How it works

  1. Request. Your agent sends a normal HTTP call to a peer's endpoint.
  2. 402 challenge. The server replies with payment requirements — asset, amount, recipient, validity window, nonce.
  3. Cap check. Local middleware compares the quote to X402_MAX_PAYMENT_PER_CALL_PTON and X402_MAX_TOTAL_SPEND_PTON. Over the cap → refuse immediately. The wallet is never asked to sign a refused payment.
  4. Sign + retry. The operator wallet signs an EIP-3009 TransferWithAuthorization over PTON; the agent retries with the base64 voucher in X-Payment.
  5. 200 receipt. Server accepts the voucher and serves the response. We emit a X402_PAYMENT event so you can audit every outbound spend.

Trust & safety

  • Caps are enforced locally, before the wallet sees the request. You can't be tricked into signing a 100 PTON voucher when your per-call cap is 1.0.
  • Each voucher is single-use. EIP-3009 nonces prevent replay; the validity window prevents long-lived signatures from sitting around.
  • No facilitator by default. We trust the upstream 2xx as proof of settlement — the same model as Tokamak's inbound billing rail. X402_FACILITATOR_URL is reserved for a future verification mode if you'd rather check settlement out-of-band.
  • Discovery is read-only. Fetching an AgentCard never spends money. You pick which skill to invoke from chat — the CALL_A2A_AGENT action runs the paid request.
